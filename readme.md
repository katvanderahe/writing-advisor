# Writing Advisor

The code for this was partially generated by Microsoft Copilot. 

## 1. Problem: how to help improve writing
  Writing is challenging and students and professionals struggle to: 
  - understand and identify the weakness within their own writing, specfically issues of clarity, structure, and tone
  - need access to clear and accurate suggestions, and it can be difficult/time consuming to try and book an appointment with a unversity's writing centre or you may not have access to these resources, especially when there is a time crunch and you need help right away
  - want concrete suggestions on how to improve

Feedback is essential to improving writing and preventing you from making the same mistakes. 

## 2. How this application helps
This application provides an easy and simple way to recieve immediate and specific feedback on the writing you submit:

- Paste your writing into the interface 
- App will send your text to a writing advice API (or local analysis function)
- it will return specfic suggestions on how to improve the clarity, structure and style
- submissions are all stored in a database so you progress can be reviewed over time 

It will turn your browser into a lightweight writing coach that will always be available to be used. 

## 3. How it works (architecture)
The app was built with Python, Flask, and SQLite. 

- The User interface 
  - Built with HTML templates in the templates/ folder 
  - `index.html` provides a text area to input and submit text
  - `history.html` shows previous analyses that stored within the database 

- Application Logic 
  - `app.py` defines the Flask application.  
  - The `/` route:
    - Handles `GET` requests to show the form.
    - Handles `POST` requests when you submit text.
    - Calls a function that contacts the writing-advice API (or a mock function) and returns suggestions.
    - Saves the original text and the advice to the database.
  - The `/history` route:
    - Queries the database for past analyses.
    - Renders them in a simple list.

-Database  
  - `models.py` defines the `WritingSample` model with:
    - `text`: the original user text.
    - `advice`: the feedback returned by the API.
    - `created_at`: timestamp of the analysis.
  - Uses SQLite via `Flask_SQLAlchemy`.  
  - The database file (`writing.db`) is created automatically in the `instance/` folder.

- API Integration  
  - The function `get_writing_advice(text)` in `app.py` is responsible for calling an external API.  
  - In this sample, it’s mocked with simple rules, but you can:
    - Add a real endpoint URL.
    - Use an API key stored in an environment variable.
    - Parse the JSON response and return the advice string.
